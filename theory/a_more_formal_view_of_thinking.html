A More Formal View of Thinking

Here, we take another look at thinking. The idea that thinking is a process is introduced.

Consider the polynomial function,

f(x) = x2 - 4.

Obviously, this is an ordinary mathematical function; however, computing its value at, say, x = 3, is a process. Specifically, it is a process over the set of all algebraic expressions.

Let p be the process of evaluating f at 3. We break the evaluation of f into three distinct steps, and consider each state i before each process takes place and after each process takes place:

p(0):

f(x) = x2 - 4;

x = 3

The next step will be to replace the chosen value for the independant variable and discard the assignment, x = 3:

p(1):

f(3) = 32 - 4

Next, we begin following the rules of order of operation, and square first:

p(2):

f(3) = 9 - 4

Finally, we subtract four and obtain the desired result:

p(3):

f(3) = 5

At this point, evaluation has concluded, and the process has ended. Some notes from this example:

First, the process of evaluating an arithmetic expression is implicitly broken down into constituent steps. In my opinion, we apply the same process (that is, our process) four distinct times, before we have completed the task. The overarching change, from p(0) to p(3) is the result of an abstract (or meta) process; we implement this process by applying a more feasible process four distinct times.

As I picture it,

Abstraction is present in many forms in this process. At every step, we focus on one part of the expression, and ignore other parts; this is an implicit form of abstraction. Replacing each expression with another which is simpler (i.e., replacing 32 with 9) is another form. In p(1), we discard the information that x=3: Because x is no longer present, this information is no longer relevant. Finally, after obtaining the result in p(3), we abstract from the computation itself: f(3) = 5 is all that we require. This is analogous to accepting the conclusion of an argument by discarding the reasoning that got us there.

Now we look at the process more literally. All processes make change. After making their change, the result of a given process (a new and distinct state) remains unchanged for some length of time. This can be understood as a "resolved" state; a process-state with nonzero duration, the lowest-energy state that a process can be in. After the resolved state, a process ceases to exist.

For instance, after obtaining the result, f(3) = 5, that result sticks around for some time, allowing the individual to use or reference it. This is true for each step of the process, as well: p(1) observes the result of p(0); p(2) observes the result of p(1), etc. Each of these "observations of the result of the previous process" are abstractions from the previous process.

Conclusion

I do not feel that I've taken any gratuitous leaps with this assessment; I feel that this is a clear portrayal of what's more or less expected of individuals as they learn elementary mathematics. It seems prudent to point out that I am not claiming that each state p(i) in the above process is a consequence of a single abstraction. Rather, I claim that the above process can be implemented via abstraction alone, by considering and casting it in different lights. We will require more theory to discuss this intelligently– in particular, processes.
