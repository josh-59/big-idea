Looking Forward

Order

A singular process need not execute in isolation. The order of a process is the number of previous instances of itself that a process observes. In particular, this affords the opportunity of self-consciousness. Actually, this could provide a very elegant approach to the idea of learning, with "memory" or "memorization" being somehow equivalent to very high order.

Efficiency

A process is learning if it gets better at what it does. This is sufficiently broad to capture the common intuitive notion of learning; our task is to consider and describe this formally.

Given that x is an arbitrary resource, we define the efficiency of a realized process p(a) with respect to some quantifiable resource x as, (p(a))/x. If p(a) uses no x, then we define the efficiency of p(a) with respect to x to be infinite.

Note that we have not precluded x from being instances of the process itself; that is, "efficacy" can be phrased in terms of efficiency. Also, for each realized process p(a), and each resource x, the efficiency of p(a) with respect to x is defined. This allows the comparison of relative efficiency between two arbitrary processes.

Learning

A process is economizing if its execution alone implies an increase in efficiency; this is my interpretation of "learning." Ultimately, learning is the first derivative of efficiency. Note that learning is a process— it is a process on the fielding process itself. As a consequence, it is subject to the above formal descriptions.

Second Derivative

The second derivative of efficiency may also gainfully be considered. This is the change in the rate of learning, and will be called cognisance. For instance, one might pick up new study habits, and thereby increase the efficiency of one's study.

Already, we have enough formal theory to make nontrivial statements about the differences in intellects around us. For instance, a common housecat is learning, but is noncongnizant. It does not plan to learn; it does not study; it does not study its study.

Clever

A concept distinct from, though related to, intelligence, a process is clever if it looks ahead. This will be phrased formally as concurrence among processes.

Creative and Optimizing

One (inelegant) approach to generating a learning process would involve a single process being alternatively creative (i.e., trying new things) and optimizing (i.e., discarding suboptimal solutions).

Schizo

A more elegant approach to generating a learning process would be to implement an optimizing function that sees the same thing in two distict lights simultaneously. Thus, each "creative" and "optimizing" would occur as a result of the same process.

Intellect

My target object is the intellect, a self-conscious, learning process of order n > 0 and depth k > 3, I believe. I think that the mind is, effectively, a data structure. Specifically, it is a data structure which is active, as opposed to static.

Moreover, I think that it is a data structure which operates in O(0) (Big-Oh-Zero) time; that is, it is infinitely economizing. As a conjecture, I think that any data structure operating in O(0) time will prove to be an intellect (and thus be an "AI" as think of them).

Consciousness

A process which can enact change upon itself is conscious.

Significance

Meaning strikes me as, "the difference it makes;" that is, significance is the change implied (by a given thing). It might be sufficient to clause significance within a conditional. My point is, if a given statement or artifact makes no change, then it has no significance, has no meaning, and is not a process.

Sincerity

Concurrence among active processes; dependant upon "order."

Physical Processes

We define a physical process as a process on physical states. By definition, the change that a process makes is nonzero, and both the argument state and the resulting state are well-defined. Therefore, a physical process has duration.
